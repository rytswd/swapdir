** 🌄 Overview

*swapdir* enables swapping a portion of the current working directory path with a replacement string, bringing the powerful ~cd old new~ functionality of ksh/zsh to all major shells (Bash, Fish, Nushell).

#+begin_example
/tmp/frontend/src/components:
❯ swapdir frontend backend

/tmp/backend/src/components:
❯ 
#+end_example

While Zsh users might know this built-in feature, *swapdir* improves upon it with:
1. *Cross-shell consistency*: Use the same workflow in Bash, Fish, Nushell, and Zsh.
2. *Smart completions*: Only suggests valid replacement paths based on your current location.
3. *Ambiguity handling*: Interactively selects paths if multiple valid swaps exist.

-----

** ✨ Features

- *Cross-Shell* — First-class support for Bash, Zsh, Fish, and Nushell.
- *Smart Completion* — Tab-complete the "old" part from your current path, and the "new" part from available siblings.
- *Static Binary* — Written in Zig for high performance and zero runtime dependencies.
- *Interactive* — Menus for resolving multiple matching paths.

** 🚀 Quick Start

*** 1. Install

**** Using Nix (Flakes)

#+begin_src nix
{
  inputs.swapdir.url = "github:rytswd/swapdir";

  outputs = { nixpkgs, swapdir, ... }: {
    # Add to your system or home-manager packages
    environment.systemPackages = [ swapdir.packages.x86_64-linux.default ];
  };
}
#+end_src

**** Manual Build (Zig)

Requirements: [[https://ziglang.org/][Zig]] 0.13+

#+begin_src bash
git clone https://github.com/rytswd/swapdir
cd swapdir
zig build -Doptimize=ReleaseSafe
cp zig-out/bin/swapdir ~/.local/bin/
#+end_src

*** 2. Setup Shell Integration

To use ~swapdir~ effectively, you need a shell wrapper function (usually aliased as ~sd~) that handles the directory change.

#+html:<details>
#+html:<summary><strong>Bash</strong></summary>

Add to =~/.bashrc=:

#+begin_src bash

# Or manually source the provided script
source /path/to/swapdir/shell/swapdir.bash
#+end_src

** 📖 Usage Guide

*** Basic Swapping

Assume you are in:
=~/work/project-v1/src/libs=

And you want to switch to the "v2" version of the project:

#+begin_src bash
sd v1 v2
#+end_src

Result:
=~/work/project-v2/src/libs=

*** Smart Completion

*swapdir* knows your current path components.

1. Type ~sd~ and press =TAB=. It will suggest parts of your current path (e.g., =work=, =project-v1=, =src=, =libs=).
2. Select ~project-v1~.
3. Press =TAB= again. It will look for sibling directories of ~project-v1~ (e.g., ~project-v2~, ~project-old~) and suggest them.

*** Handling Ambiguity

If a swap results in multiple valid paths (e.g., swapping "src" to "test" matches in two places), *swapdir* will prompt you to choose:

#+begin_src text
Multiple valid paths found:
  1) /home/user/lib/project/test
  2) /home/user/src/project/test
Select [1-2]:
#+end_src


** 🔧 How It Works

*swapdir* follows the [[https://clig.dev/][CLI Guidelines]] philosophy: separate the logic from the shell state.

1. The shell function ~sd~ calls the ~swapdir~ binary.
2. ~swapdir~ calculates the new path and prints it to stdout.
3. If successful, the shell function captures the output and runs ~cd~.
4. If ambiguous, the shell function presents the menu.
