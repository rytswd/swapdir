** 🌄 Overview

*swapdir* enables swapping a portion of the current working directory path with a replacement string, bringing the powerful ~cd old new~ functionality of ksh/zsh to all major shells (Bash, Zsh, Fish, and Nushell).

#+begin_example
/tmp/frontend/src/components:
❯ sd frontend backend

/tmp/backend/src/components:
❯ 
#+end_example

While Zsh users might know this built-in feature, *swapdir* improves upon it with:
1. *Cross-shell consistency*: Use the same workflow in Bash, Fish, Nushell, and Zsh.
2. *Smart completions*: Only suggests valid replacement paths based on your current location.
3. *Ambiguity handling*: Interactively selects paths if multiple valid swaps exist.

-----

** ✨ Features

- *Cross-Shell* — First-class support for Bash, Zsh, Fish, and Nushell.
- *Smart Completion* — Tab-complete the "old" part from your current path, and the "new" part from available siblings.
- *Static Binary* — Written in Zig for high performance and zero runtime dependencies.
- *Interactive* — Menus for resolving multiple matching paths.

** 🚀 Quick Start

*** 1. Install

**** Home Manager (recommended)

The flake ships a Home Manager module that installs the binary *and* wires up the ~sd~ wrapper function and tab completions for you.

#+begin_src nix
{
  inputs.swapdir.url = "github:rytswd/swapdir";

  # In your Home Manager configuration:
  imports = [ inputs.swapdir.homeModules.swapdir ];
  programs.swapdir.enable = true;
}
#+end_src

Bash, Zsh, and Fish integration are enabled by default; Nushell is opt-in. Toggle any of them explicitly:

#+begin_src nix
programs.swapdir = {
  enable = true;
  enableBashIntegration = true;   # default: true
  enableZshIntegration = true;    # default: true
  enableFishIntegration = true;   # default: true
  enableNushellIntegration = true; # default: false
};
#+end_src

With the module enabled you can skip step 2 entirely.

**** Nix package (without the module)

Install just the binary (for example into ~environment.systemPackages~ or ~home.packages~), then set up the shell integration manually as shown in step 2:

#+begin_src nix
environment.systemPackages = [
  inputs.swapdir.packages.x86_64-linux.swapdir
];
#+end_src

The shell integration scripts ship inside the package at =$out/share/swapdir/= (e.g. =~/.nix-profile/share/swapdir/swapdir.bash=).

**** Manual Build (Zig)

Requirements: [[https://ziglang.org/][Zig]] 0.17 (development/master — 0.17 is not yet released).

#+begin_src bash
git clone https://github.com/rytswd/swapdir
cd swapdir
zig build -Doptimize=ReleaseSafe
cp zig-out/bin/swapdir ~/.local/bin/
#+end_src

The shell integration scripts live under =shell/= in the repository.

*** 2. Setup Shell Integration

/Skip this step if you used the Home Manager module — it does this for you./

To use ~swapdir~ effectively you need a shell wrapper function (aliased as ~sd~) that captures the binary's output and runs ~cd~. Source the script for your shell from wherever it is installed — either the package's =share/swapdir/= directory or the repository's =shell/= directory.

#+html:<details>
#+html:<summary><strong>Bash</strong></summary>

Add to =~/.bashrc=:

#+begin_src bash
source ~/.nix-profile/share/swapdir/swapdir.bash
# Or, from a manual build / checkout:
# source /path/to/swapdir/shell/swapdir.bash
#+end_src

#+html:</details>

#+html:<details>
#+html:<summary><strong>Zsh</strong></summary>

Add to =~/.zshrc=:

#+begin_src zsh
source ~/.nix-profile/share/swapdir/swapdir.zsh
# Or: source /path/to/swapdir/shell/swapdir.zsh
#+end_src

#+html:</details>

#+html:<details>
#+html:<summary><strong>Fish</strong></summary>

Add to =~/.config/fish/config.fish=:

#+begin_src fish
source ~/.nix-profile/share/swapdir/swapdir.fish
# Or: source /path/to/swapdir/shell/swapdir.fish
#+end_src

#+html:</details>

#+html:<details>
#+html:<summary><strong>Nushell</strong></summary>

Add to your config:

#+begin_src nu

# Or: source /path/to/swapdir/shell/swapdir.nu

# Or: . /path/to/swapdir/shell/swapdir.sh
#+end_src

Note: completions are not available under POSIX sh.

** 📖 Usage Guide

*** Basic Swapping

Assume you are in:
=~/work/project-v1/src/libs=

And you want to switch to the "v2" version of the project:

#+begin_src bash
sd v1 v2
#+end_src

Result:
=~/work/project-v2/src/libs=

*** Smart Completion

*swapdir* knows your current path components.

1. Type ~sd~ and press =TAB=. It will suggest parts of your current path (e.g., =work=, =project-v1=, =src=, =libs=).
2. Select ~project-v1~.
3. Press =TAB= again. It will look for sibling directories of ~project-v1~ (e.g., ~project-v2~, ~project-old~) and suggest them.

*** Handling Ambiguity

If a swap results in multiple valid paths (e.g., swapping "src" to "test" matches in two places), *swapdir* will prompt you to choose:

#+begin_src text
Multiple valid paths found:
  1) /home/user/lib/project/test
  2) /home/user/src/project/test
Select [1-2]:
#+end_src


** 🔧 How It Works

*swapdir* follows the [[https://clig.dev/][CLI Guidelines]] philosophy: separate the logic from the shell state.

1. The shell function ~sd~ calls the ~swapdir~ binary.
2. ~swapdir~ calculates the new path and prints it to stdout.
3. If successful, the shell function captures the output and runs ~cd~.
4. If ambiguous, the shell function presents the menu.
